What is SVPR?

SmartPhoneVR is a project to allow you to use your Android smartphone / mobile phone as a cheap HMD. You can already find Google Cardboard games, which run directly on your device. SVPR is used to play games on your computer, which use the OpenVR / SteamVR API. That is, almost all games or applications, that can be used with the HTC Vive.

Warning!

I did this for fun! I plan to continue working on it, but it will never provide a "true VR experience". Do not expect anything... I want to make sure, that you don't expect anything close to an real HMD! Simulation sickness is real, e.g., rotating your head will make you dizzy. The image quality is low (i.e., artifacts and low resolution), like when you forgot your glasses. On the other side, the 3D experience works, which is kinda cool.

How does it work?

The Android app reads out your phones sensor data and forwards them over network (currently udp) to the Control instance on your PC, which further directs it to the driver instance in the game / application.

How can I build and use SVPR?

Assuming, you know your way around C++ on windows and how to build an android application... you need

  • CMake
  • Visual Studio, tested with 2013 and 2015
  • Boost, tested with 1.60
  • glm
  • Android Studio
  • Steam + SteamVR
First checkout svpr.control from github with submodules, and, for later, svpr.android.

Copy the glm headers into driver/.

After compiling svpr.driver and svpr.control, the resulting driver_svpr.dll needs to be copied into PathTo/steamapps/common/SteamVR/drivers/spvr/bin/win32/driver_spvr.dll, or .../win64/..., respectively.

You can now start the svpr.control (allow network access) and run the SteamVR compositor with PathTo/steamapps/common/SteamVR/bin/win32/vrcompositor.exe -keepalive -debug.

After building and running the android app, you can set the IP. svpr.control should now print the received sensor data. The compositor's rendering should move with your mobile phone.
